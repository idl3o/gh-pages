StreamPaymentContract

Solidity 0.8.17 Audited ERC-20 Compatible Upgradeable

The StreamPaymentContract manages all payment operations for the Web3 Crypto Streaming platform. It handles subscription payments, one-time content purchases, and automated revenue distribution to creators, platform, and other stakeholders.

This contract is designed to work seamlessly with the StreamAccessContract to automatically grant viewing rights upon successful payment processing, creating a frictionless user experience while maintaining full decentralization and transparency.

Key Features

  • Support for multiple payment tokens (ETH, USDC, DAI, platform token)
  • Subscription management with recurring payments
  • Pay-per-view content purchases
  • Automated revenue distribution
  • Refund mechanisms for disputed content
  • Integration with StreamAccessContract for seamless viewing rights
  • On-chain payment verification and receipt generation

Functions

Payment Processing

purchaseContent(bytes32 contentId, uint256 duration) payable → bool

Processes a one-time payment for content access. Accepts ETH payment directly and grants access through the StreamAccessContract for the specified duration.

Parameters

Name Type Description
contentId bytes32 The unique identifier of the content being purchased
duration uint256 Duration of access in seconds

Returns

bool: True if the purchase was successful

Example

// Purchase access to content for 7 days
function purchaseWeeklyAccess(bytes32 contentId) public payable {
    // 7 days in seconds
    uint256 weekDuration = 7 * 24 * 60 * 60;
    
    // Purchase content with ETH
    streamPaymentContract.purchaseContent{value: 0.01 ether}(contentId, weekDuration);
}

purchaseContentWithToken(address token, uint256 amount, bytes32 contentId, uint256 duration) → bool

Processes a one-time payment using ERC-20 tokens for content access. Requires prior approval for the contract to spend tokens on behalf of the user.

Parameters

Name Type Description
token address The address of the ERC-20 token contract
amount uint256 Amount of tokens to pay
contentId bytes32 The unique identifier of the content being purchased
duration uint256 Duration of access in seconds

Returns

bool: True if the purchase was successful

Subscription Management

Admin Functions

Events

The contract emits the following events to track payment activities:

  • PaymentProcessed(address indexed payer, bytes32 indexed contentId, uint256 amount, uint256 duration)
  • SubscriptionCreated(address indexed subscriber, uint256 indexed planId, uint256 startTime)
  • SubscriptionRenewed(address indexed subscriber, uint256 indexed planId, uint256 newExpiration)
  • SubscriptionCancelled(address indexed subscriber, uint256 indexed planId, uint256 endTime)
  • RevenueDistributed(address indexed creator, uint256 creatorAmount, uint256 platformAmount)
