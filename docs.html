Whitepaper

Download Whitepaper (PDF)

Abstract

StreamChain represents a paradigm shift in content distribution. By leveraging blockchain technology, we have created a truly decentralized streaming platform that eliminates intermediaries, provides direct monetization for creators, and returns content ownership and control to its rightful owners. This document outlines the technical architecture, token economics, and governance model of the StreamChain platform.

Problem Statement

Centralized streaming platforms have created numerous issues for content creators and consumers:

  • Creators receive only a fraction of generated revenue
  • Platform censorship limits creative freedom
  • Opaque recommendation algorithms create content echo chambers
  • User data privacy concerns abound with targeted advertising
  • Content is vulnerable to platform policy changes and shutdowns

Our Solution

StreamChain provides a decentralized alternative that addresses these issues through:

  • Direct creator-to-viewer payments via smart contracts
  • Content stored on IPFS for censorship resistance
  • Transparent recommendation algorithms governed by community
  • User privacy preserved through anonymous viewing options
  • True content ownership through blockchain verification

Architecture

StreamChain's architecture consists of several interconnected components:

Decentralized Storage Layer

Content is stored on IPFS (InterPlanetary File System), a distributed storage protocol. When a creator uploads content, it's split into chunks, encrypted if specified, and distributed across the IPFS network. This ensures content cannot be censored or removed by a single entity.

Blockchain Layer

Content metadata, ownership records, and payment instructions are stored on the blockchain. Smart contracts manage the relationship between creators, viewers, and content, automating payments and ensuring transparency.

Application Layer

The web and mobile interfaces provide a user-friendly experience for creators and viewers. This layer interacts with both the blockchain and IPFS through dedicated APIs and SDKs.

API Reference

Our API allows developers to integrate StreamChain functionality into their applications.

Authentication

All API requests require authentication using JWT tokens or Web3 wallet signatures. 


// Example Authentication Request
const authResponse = await fetch('https://api.streamchain.io/v1/auth', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    signature: signedMessage,
    wallet: userWalletAddress
  })
});

const { token } = await authResponse.json();

Content Endpoints

These endpoints allow interaction with the content library.

GET /v1/content

Retrieves a paginated list of available content

Parameters:
  • page (optional): Page number, default 1
  • limit (optional): Items per page, default 20
  • category (optional): Filter by content category

GET /v1/content/:id

Retrieves details about a specific content item

POST /v1/content

Uploads new content to the platform

User Endpoints

These endpoints allow management of user accounts and profiles.

GET /v1/users/profile

Retrieves the profile of the authenticated user

PUT /v1/users/profile

Updates the profile of the authenticated user

GET /v1/users/:walletAddress

Retrieves the public profile of a user by wallet address

Payment Endpoints

These endpoints handle payment operations and transactions.

GET /v1/payments/history

Retrieves payment history for the authenticated user

Parameters:
  • page (optional): Page number, default 1
  • limit (optional): Items per page, default 20
  • type (optional): Filter by payment type (subscription, tip, etc.)

POST /v1/payments/tip

Sends a tip to a content creator

POST /v1/payments/subscribe

Creates a subscription to a creator's channel

Smart Contracts

StreamChain's core functionality is powered by a suite of smart contracts that handle content ownership, payments, and platform governance.

Content Registry Contract

This contract manages the registration and ownership of content on the platform. It implements ERC-721 for non-fungible content tokens and includes mechanisms for:

  • Content registration and minting
  • Ownership verification and transfers
  • Content metadata storage
  • Access control for permissioned content

// Example content registration
await contentRegistryContract.methods.registerContent(
  contentId,
  metadataHash,
  accessControl,
  royaltyPercentage
).send({ from: creatorWalletAddress });

Payment Contract

Handles all payment-related operations including tips, subscriptions, and revenue sharing. Key features include:

  • Direct creator payments
  • Subscription management
  • Automated revenue distribution
  • Fee calculation and platform revenue

Governance Contract

Enables community-based decision making for platform parameters and upgrades. Leverages the STREAM token for voting power.

SDK Documentation

Our Software Development Kits (SDKs) provide easy integration with the StreamChain platform for various programming languages and environments.

JavaScript SDK

The JavaScript SDK is the most comprehensive, supporting both browser and Node.js environments. It provides full access to the StreamChain API and smart contract functionality. 


// Example JavaScript SDK usage
import { StreamChain } from '@streamchain/sdk';

// Initialize the SDK
const streamChain = new StreamChain({
  apiKey: 'YOUR_API_KEY',
  web3Provider: window.ethereum
});

// Connect user's wallet
await streamChain.connect();

// Get content list
const content = await streamChain.content.list({
  category: 'gaming',
  page: 1,
  limit: 10
});

Python SDK

Ideal for backend integrations, data analysis, and machine learning applications related to the StreamChain platform. 


# Example Python SDK usage
from streamchain import StreamChain

# Initialize the SDK
stream_chain = StreamChain(
    api_key="YOUR_API_KEY",
    private_key="YOUR_WALLET_PRIVATE_KEY"
)

# Get user profile
profile = stream_chain.users.get_profile()

# Upload content
content_id = stream_chain.content.upload(
    file_path="./video.mp4",
    title="My Web3 Video",
    description="A video about Web3 streaming",
    tags=["web3", "blockchain", "streaming"]
)

Installation

Our SDKs can be installed via standard package managers:


# JavaScript/Node.js
npm install @streamchain/sdk

# Python
pip install streamchain-sdk
